- touch = Creacion de archivos nuevos.
- mkdir = Creacion de nueas carpetas.
- git init = Inicio del repositorio local.
- pwd = ruta en donde estás.
- cat Agu.txt = Muestra contenido de un archivo.
- sudo = Dar permiso de administrador.
- ls = Lista de carpetas que hay.
- git status = Muestra los archivos que fueron modificados y no incluidos en el repo.
- git add . = Agrga todos los archivos juntos.
- git commit -m =para hacer un commit general o particular de cada archivo.
- ls -al = Me hace una lista de todos los archivos, inclusive los ocultos.
- git log Agu.txt  = Me muestra los commits de Agu.txt .
- git show Agu.txt = Muestra cambiaos en el archivo.
- git checkout dafewr8239rufenwc = te muestra y vuelve a esa version del archivo. (si no hago el commit, puedo volver para atrás, sinó, me superpone esa version como final)
- git checkout master e13o2r435tjk324 = vuelvo a la version esa .
- git rm --force Agu.txt = Elimina el archivo ese de git y del HDDD.
- git reset Agu.txt = vuelve en el timepo per no podemos ir,  mirar y volver como checkout, sio que este vuelve pero reescribe todo y no hay vuelta aatrás.
- git reset --hard Agu.txt = borra todo en el area de staging y pierde todo para siempre.
- git reset --soft Agu.txt =mantiene ahí nuestos archivos del area de staging para q podamos aplicar nuestros ultimos cambios pero desde un commmit anterior  (es mas seguro).
- git reset head Agu.txt = Saca los archivos del area de staging . No para borrarlos, solo para q los ultimos cambios de estos archivos no se envíen al último commit. A menos q cambiemos de opiñon y los incluyamos con git add. 
Ej de reses head: si añadimos todos los archivos a staging y resulta q uno lo necesitamos modificar antes,sólo queremos sacar 1. Ejecutamos el comando y sólo  se van a mover de staging a unstaged.
